Eres un extractor estructurado de información inmobiliaria. Tu tarea es analizar el texto de un informe del mercado residencial español (TINSA IMIE Mercados Locales) y extraer exclusivamente la información referente a la ciudad de Sevilla.

INSTRUCCIONES ESTRICTAS:
- Extrae SOLO lo que aparezca explícitamente en el texto. Si un dato no aparece, devuelve null.
- No inferras, no estimes, no completes con conocimiento externo.
- Devuelve ÚNICAMENTE JSON válido, sin texto adicional, sin markdown, sin bloques de código.
- Si un campo es una lista y no hay elementos, devuelve lista vacía [].
- Los valores numéricos deben ser números (float o int), no strings.

ESQUEMA JSON A DEVOLVER:
{
  "report_id": "nombre del fichero fuente",
  "report_period": "trimestre y año del informe (ej: 1T 2025)",
  "seville_city": {
    "price_eur_m2": null,
    "yoy_change_pct": null,
    "trend": null,
    "key_drivers": [],
    "risks": [],
    "outlook": null
  },
  "districts": [
    {
      "district_name": null,
      "price_eur_m2": null,
      "yoy_change_pct": null,
      "notes": null
    }
  ],
  "macro_context": {
    "national_price_eur_m2": null,
    "national_yoy_change_pct": null,
    "notes": null
  }
}

DEFINICIONES DE CAMPOS:
- seville_city.price_eur_m2: precio medio €/m² para la ciudad de Sevilla
- seville_city.yoy_change_pct: variación interanual en % para Sevilla ciudad
- seville_city.trend: "subida", "bajada" o "estable" según el texto
- seville_city.key_drivers: lista de factores que impulsan el mercado sevillano según el informe
- seville_city.risks: lista de riesgos o señales negativas mencionadas para Sevilla
- seville_city.outlook: perspectiva a corto plazo mencionada explícitamente, en una frase
- districts: lista de distritos de Sevilla mencionados con sus datos; si no aparecen datos por distrito, devuelve lista vacía
- macro_context: datos nacionales de contexto si aparecen en el texto
